# traffic-top.sh — Linux 网络流量使用排行查询工具

一个纯 Bash 编写的 Linux 网络流量查询脚本，支持按进程、网络接口、连接三个维度排行，以及类似 `top` 的实时监控模式。

## 特性

- 📊 **多维度排行** — 按进程 / 接口 / 连接 / 全部
- 🎨 **彩色终端输出** — Top 3 高亮，直观易读
- 📈 **流量占比** — 自动计算百分比
- 📦 **人类可读格式** — B / K / M / G 自动转换
- 🔄 **实时监控** — 类似 `top` 的持续刷新
- 🛡️ **安全执行** — `set -euo pipefail`
- 📝 **零外部依赖** — 仅需 `ss`, `awk`, `sort`, `bc`（Linux 预装）

## 安装

```bash
chmod +x traffic-top.sh
# 直接运行
./traffic-top.sh
```

## 用法

```
traffic-top.sh [选项]
```

### 选项

| 选项 | 说明 | 默认值 |
|------|------|--------|
| `-m, --mode <type>` | 排行模式：`process` / `interface` / `connection` / `all` | `process` |
| `-n, --count <num>` | 显示前 N 条结果 | `10` |
| `-t, --top <num>` | 同 `--count` | `10` |
| `-d, --delay <sec>` | 刷新间隔秒数（仅实时监控模式） | `2` |
| `-c, --cumulative` | 累计模式（仅 process 模式） | — |
| `-i, --include-lo` | 包含 loopback 接口 | — |
| `-h, --help` | 显示帮助信息 | — |

## 使用示例

### 按进程排行（默认）

```bash
./traffic-top.sh
./traffic-top.sh -m process
./traffic-top.sh -m process -n 5
```

```bash
./traffic-top.sh -m interface
```


### 按连接排行（需 root）

```bash
sudo ./traffic-top.sh -m connection
sudo ./traffic-top.sh -m connection -n 20
```

```

================================================================================

  #2       10.0.0.50:80                   89       ESTABLISHED
  #3       172.16.0.1:22                  12       ESTABLISHED
  ...

  连接总数: 256
```

### 同时显示所有模式

```bash
./traffic-top.sh -m all
```

### 实时监控模式

```bash
# 按进程实时刷新，每 1 秒更新一次
./traffic-top.sh -m process -n 5 -d 1

# 按接口实时刷新
./traffic-top.sh -m interface -d 2
```

## 输出说明

### 各模式字段含义

**进程模式：**
- **排名** — 按总流量降序排列
- **进程 (PID)** — 进程名和进程 ID
- **接收流量 / 发送流量 / 总流量** — 累计字节数
- **占比** — 占系统总流量的百分比

**接口模式：**
- **接口** — 网络接口名称（如 `enp3s0`, `docker0`）
- **接收 / 发送 / 总计** — 各接口累计流量
- **收占比 / 发占比** — 该接口收发流量占系统总量的比例

**连接模式：**
- **远程地址** — 远程 IP:端口
- **连接数** — 该地址的活跃连接数
- **状态** — TCP 连接状态（ESTABLISHED / TIME_WAIT 等）

### 实时监控模式

实时监控模式采用**差值计算**：
1. 采集第一次快照
2. 等待指定间隔（`-d`）
3. 采集第二次快照
4. 计算差值，显示为 **速率**（如 `12.3M/s`）
5. 循环刷新，按 `Ctrl+C` 退出

## 注意事项

- **进程排行**和**连接排行**需要 root 权限才能获取完整信息
- 进程流量基于 `/proc/<pid>/net/dev` 读取，反映该进程所属命名空间的累计流量
- 实时监控模式在后台容器或某些命名空间环境中可能行为异常
- 脚本使用 `bc` 进行浮点运算，确保系统已安装
